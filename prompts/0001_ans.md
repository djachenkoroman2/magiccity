Ниже описываю современную генерацию Minecraft Java Edition, в основном 1.18+ и новее. В Bedrock идеи похожи, но реализация отличается. До 1.18 мир генерировался заметно иначе: рельеф сильнее зависел от биома, а пещеры чаще “вырезались” отдельными алгоритмами.

**Главная идея**
Minecraft не хранит заранее весь мир. Он хранит `seed`, настройки измерения и уже посещенные чанки. Когда игрок подходит к новой области, игра генерирует чанки размером `16x16` блоков по X/Z и всей высотой мира.

Генерация детерминирована:

```text
seed + координаты чанка + версия игры + настройки мира => одинаковый результат
```

Но результат не является “просто случайным”. Используются процедурные шумы: Perlin-like / gradient noise, octave noise, density functions, climate maps, surface rules и placed features.

**1. Seed И Псевдослучайность**
Seed превращается в набор псевдослучайных потоков. Разные системы получают свои “соли”:

```text
один поток для биомов
другой для структур
другой для руд
другой для деревьев
...
```

Поэтому деревья, руды и деревни выглядят случайно, но при том же seed появляются в тех же местах.

Простая случайность дала бы хаос: каждый блок был бы независим. Minecraft вместо этого использует связный шум, где соседние координаты дают близкие значения. Это создает плавные горы, равнины, океаны и долины.

**2. Шумы И Октавы**
Базовый алгоритм похож на это:

```text
noise(x, z) -> значение от -1 до 1
```

Низкочастотный шум дает крупные формы: материки, океаны, большие горные зоны.

Высокочастотный шум дает детали: неровности, уступы, шероховатость склонов.

Несколько шумов складываются в “октавы”:

```text
value =
  noise(x * 0.001, z * 0.001) * 1.0   // крупные формы
+ noise(x * 0.01,  z * 0.01)  * 0.5   // средние формы
+ noise(x * 0.05,  z * 0.05)  * 0.2   // детали
```

Так получается природный фрактальный вид: большие структуры содержат меньшие детали.

**3. Климат И Биомы**
Современный Overworld использует многомерную климатическую систему. Для каждой позиции считаются параметры:

```text
temperature     // температура
humidity        // влажность / vegetation
continentalness // океан, берег, суша, глубина материка
erosion         // сглаженность или гористость
weirdness       // долины, пики, странные варианты ландшафта
depth           // поверхность или подземные биомы
```

Биом выбирается не просто по карте “тут пустыня, тут лес”. Игра берет точку в 6-мерном пространстве:

```text
P = (temperature, humidity, continentalness, erosion, weirdness, depth)
```

У каждого биома есть диапазоны этих параметров. Если точка попала в диапазон, выбирается этот биом. Если не попала точно, выбирается ближайший подходящий биом.

Примерно:

```text
низкая temperature + высокая continentalness => snowy plains / taiga / frozen peaks
высокая temperature + низкая humidity        => desert / savanna
высокая humidity + тепло                     => jungle
низкая continentalness                       => ocean
средняя continentalness                      => coast / beach
```

`depth` позволяет делать 3D-биомы: например, под обычным лесом может быть lush cave или dripstone cave.

**4. Continentalness, Erosion, Weirdness**
Эти параметры важны не только для биомов, но и для формы мира.

`continentalness` отвечает за “насколько мы внутри материка”:

```text
очень низко  => deep ocean
низко        => ocean
около нуля   => coast / beach
высоко       => inland terrain
```

`erosion` влияет на плоскость и высоту:

```text
высокая erosion => более плоские области
низкая erosion  => более резкий, гористый рельеф
```

`weirdness` преобразуется в значение peaks and valleys. Упрощенно:

```text
PV = 1 - abs(3 * abs(weirdness) - 2)
```

PV помогает определить, будет ли область долиной, низиной, обычным рельефом, высоким хребтом или пиком.

**5. Генерация Формы Через Density Functions**
Современный Minecraft не сначала рисует карту высот, а потом заполняет блоки. Он считает 3D-поле плотности:

```text
density(x, y, z)
```

Если значение больше нуля, блок считается твердым:

```text
density > 0 => stone / deepslate / netherrack / end stone
density <= 0 => air или fluid
```

Упрощенная модель:

```text
base_height = f(continentalness, erosion, weirdness)
detail      = small_noise(x, y, z)
caves       = cave_noise(x, y, z)

density = base_height - y + detail - caves
```

На практике формулы сложнее. В Java Edition они описываются через `noise_router` и `density_function`. Есть функции для континентов, эрозии, гребней, пещер, аквиферов, рудных жил и других систем.

Важно: игра не обязана считать сложную функцию для каждого блока полностью независимо. Она часто семплирует шум на более редкой сетке, а между точками интерполирует значения. Это сильно ускоряет генерацию.

**6. Noise Step: Базовый Камень, Воздух И Жидкости**
На этапе `noise` создается основная масса мира:

```text
где будет камень
где будет воздух
где будет вода
где будет лава
где будут большие пещерные объемы
```

В Overworld базовый твердый блок обычно `stone`, ниже постепенно появляется `deepslate`. В Nether это `netherrack`, в End - `end stone`.

**7. Пещеры**
После 1.18 есть несколько типов пещерной генерации.

Noise caves:

```text
cheese caves    // большие полости, как дырки в сыре
spaghetti caves // длинные извилистые тоннели
noodle caves    // более узкие и тесные тоннели
```

Они создаются через 3D-шумы, которые уменьшают плотность. Если cave noise достаточно сильный, твердое тело превращается в пустоту.

Также остаются carvers:

```text
cave carver   // ветвящиеся тоннели и комнаты
canyon carver // длинные овраги
```

Carver работает более геометрически: выбирает стартовую точку, направление, радиус, затем “вырезает” эллипсоидные объемы по траектории.

**8. Аквиферы**
Аквиферы решают, будет ли пустота заполнена водой, лавой или воздухом.

До 1.18 пещеры ниже уровня моря часто просто заливались бы водой. Аквиферы добавили локальные уровни жидкости:

```text
в одной пещере вода на Y=30
рядом сухая пещера
еще ниже лавовое озеро
```

Алгоритмически это похоже на локальную карту уровней жидкости, основанную на шуме и псевдослучайных точках. Если пещера ниже локального уровня воды, она заполняется водой. В глубинах ниже `y=0` иногда появляются лавовые аквиферы.

**9. Surface Rules: Поверхностные Блоки**
После того как форма мира готова, игра заменяет верхние слои.

Сначала мир может быть просто каменным. Затем surface rules решают:

```text
grass_block сверху, dirt ниже
sand на пляже
gravel на дне океана
snow/ice в холодных местах
terracotta в badlands
mycelium на mushroom fields
mud/mangrove blocks в болотах
```

Правила учитывают:

```text
биом
высоту Y
уровень моря
наличие воды сверху
уклон
локальный шум поверхности
температуру
```

Например, пляж возникает не потому, что “там поставили песок вручную”, а потому что область около уровня моря, рядом с океаном, с подходящим биомом и surface rule выбрала песок.

**10. Структуры**
Структуры генерируются отдельной системой:

```text
villages
strongholds
mineshafts
desert temples
ocean monuments
ancient cities
trial chambers
woodland mansions
```

Обычно процесс такой:

```text
1. По seed и координатам регионов выбираются кандидаты.
2. Проверяются spacing, separation, salt и allowed biomes.
3. Создается structure start.
4. Структура собирается из pieces или jigsaw-шаблонов.
5. Части структуры размещаются в нужных чанках.
```

Например, деревни используют набор шаблонов зданий и дорог, которые соединяются по правилам. Stronghold размещается иначе: кольцами вокруг центра мира.

**11. Features: Руды, Деревья, Озера, Растения**
После поверхности идут decorations / features.

Примеры:

```text
ores
amethyst geodes
trees
flowers
grass
kelp
cacti
bamboo
springs
lava lakes
ice patches
clay/sand/gravel disks
```

Каждая feature имеет две части:

```text
configured_feature // что именно генерировать
placed_feature     // где и как часто пытаться размещать
```

Например, руда:

```text
тип: diamond_ore
размер жилы: N блоков
количество попыток на чанк: K
диапазон высот: от Y1 до Y2
условие: заменять только stone/deepslate
```

Упрощенный алгоритм рудной жилы:

```text
выбрать случайную точку в чанке
выбрать направление и длину
создать несколько пересекающихся эллипсоидов
заменить подходящие блоки на ore
```

Большие рудные жилы в новых версиях используют отдельные шумы, которые формируют длинные подземные структуры.

**12. Деревья И Растительность**
Деревья тоже не являются заранее готовыми объектами в простом смысле. У них есть генераторы:

```text
trunk placer   // форма ствола
foliage placer // форма листвы
root placer    // корни, если нужны
decorators     // лианы, пчелиные ульи, какао и т.п.
```

Биом задает, какие деревья разрешены и с какой частотой. Поэтому jungle плотный, desert почти пустой, а taiga получает хвойные деревья.

**13. Heightmaps**
Во время генерации игра строит карты высот:

```text
WORLD_SURFACE
OCEAN_FLOOR
MOTION_BLOCKING
```

Они нужны, чтобы быстро отвечать на вопросы вроде:

```text
где верхний твердый блок?
куда поставить дерево?
где спавнить моба?
где поверхность для структуры?
```

Без heightmap пришлось бы постоянно сканировать колонку блоков сверху вниз.

**14. Свет И Финализация**
Потом считаются свет и начальный спавн некоторых мобов. Чанк проходит статусы примерно так:

```text
empty
structure_starts
structure_references
biomes
noise
surface
carvers
features
initialize_light
light
spawn
full
```

После `full` чанк становится полноценным и может быть загружен игроком.

**Очень Коротко В Псевдокоде**

```text
for each chunk:
    rng = random(seed, chunk_x, chunk_z)

    generate_structure_starts()
    calculate_biomes_using_climate_noise()

    for each block position (x, y, z):
        climate = sample_climate(x, y, z)
        density = sample_density_functions(x, y, z, climate)

        if density > 0:
            place_default_solid_block()
        else:
            place_air_or_fluid_using_aquifer()

    apply_surface_rules()
    run_carvers()
    place_structures()
    place_features_and_ores()
    calculate_heightmaps()
    calculate_lighting()
    spawn_initial_mobs()
```

**Главный Вывод**
Minecraft-ландшафт получается из нескольких наложенных систем: крупные шумы создают материки и океаны, климатические параметры выбирают биомы, 3D density functions формируют горы и пещеры, surface rules превращают камень в траву, песок и снег, а features добавляют деревья, руды, растения и структуры. Поэтому мир выглядит естественным: он не случайный по блокам, а построен из плавных, согласованных процедурных полей.
