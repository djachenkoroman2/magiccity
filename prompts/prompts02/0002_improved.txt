Необходимо опционально реализовать режим mobile LiDAR scanning для генерации облака точек с учетом лучей, траектории сенсора и окклюзий.

Требования к функциональности:

1. Режим mobile LiDAR должен быть отдельной настраиваемой опцией генерации:
   - полностью выключен по умолчанию;
   - включается через отдельный параметр конфигурации;
   - не должен ломать существующий surface sampling;
   - должен уметь работать совместно с дорогами, зданиями, участками, ограждениями и рельефом.

2. Необходимо описать модель мобильного сенсора:
   - положение сенсора в пространстве `x/y/z`;
   - высоту установки над землей;
   - направление движения;
   - траекторию движения по дорогам или заданной линии;
   - шаг между позициями сканирования;
   - скорость или плотность прохода, если это влияет на количество лучей.

3. Необходимо реализовать генерацию LiDAR-лучей:
   - горизонтальный угол обзора;
   - вертикальный угол обзора;
   - количество каналов или вертикальных слоев;
   - угловой шаг по горизонтали;
   - максимальную дальность луча;
   - минимальную дальность луча;
   - детерминированную генерацию лучей от `seed`.

4. Каждый луч должен взаимодействовать со сценой:
   - искать ближайшее пересечение с объектами сцены;
   - учитывать рельеф, дороги, тротуары, здания, крыши, фасады и ограждения;
   - возвращать только ближайшую видимую точку по направлению луча;
   - не создавать точки за объектом, если перед ним есть окклюзия.

5. Необходимо реализовать окклюзии:
   - здания должны закрывать объекты за ними;
   - заборы и стены должны частично или полностью закрывать видимость в зависимости от типа и прозрачности;
   - рельеф может перекрывать низкие лучи;
   - объекты за ближайшим пересечением луча не должны попадать в итоговое облако точек;
   - для прозрачных ограждений допускается частичное прохождение лучей через проемы или редкую структуру.

6. Нужно предусмотреть параметры шума и реалистичности:
   - случайное отклонение дальности;
   - угловой jitter;
   - вероятность пропуска луча;
   - зависимость плотности точек от расстояния;
   - ограничение по максимальной дальности;
   - возможность отключить шум для детерминированных тестов.

7. Результат mobile LiDAR mode должен отличаться от обычного surface sampling:
   - точки должны появляться только там, куда реально попали лучи;
   - дальние и закрытые поверхности должны быть прорежены или отсутствовать;
   - вертикальные фасады, заборы и ближние объекты должны получать более характерный LiDAR-паттерн;
   - за объектами должны появляться зоны тени и неполного покрытия.

8. Необходимо сохранить semantic class и RGB:
   - каждая точка должна получать класс объекта, в который попал луч;
   - `ground`, `road`, `sidewalk`, `road_median`, `building_facade`, `building_roof`, `fence`, `fence_foundation` должны сохранять свои class ids;
   - при необходимости можно добавить metadata о типе источника точки: surface sampling или mobile LiDAR.

9. Конфигурация должна позволять управлять режимом:
   - включение и выключение mobile LiDAR;
   - высота сенсора;
   - дальность сканирования;
   - горизонтальный и вертикальный FOV;
   - количество каналов;
   - плотность лучей;
   - шаг траектории;
   - параметры шума;
   - использование окклюзий;
   - возможность писать только LiDAR-точки или смешивать их с обычным surface sampling.

10. Metadata должна содержать диагностическую информацию:
    - включен ли mobile LiDAR mode;
    - количество позиций сенсора;
    - количество выпущенных лучей;
    - количество успешных попаданий;
    - количество лучей, ушедших за максимальную дальность;
    - распределение попаданий по semantic class;
    - параметры сканирования после применения значений по умолчанию.

11. Реализация должна быть детерминированной:
    - одинаковый `seed` и одинаковая конфигурация дают одинаковые лучи и точки;
    - изменение `seed` меняет jitter, пропуски и детализацию;
    - тесты должны проверять воспроизводимость.

12. Необходимо добавить тесты:
    - конфигурация mobile LiDAR загружается и валидируется;
    - при выключенном режиме старое поведение не меняется;
    - при включенном режиме появляются LiDAR-точки;
    - окклюзии не позволяют лучам создавать точки за ближайшей преградой;
    - metadata содержит статистику лучей и попаданий;
    - генерация детерминирована при одинаковом `seed`.

Итоговая реализация должна добавить реалистичный mobile LiDAR mode: генератор выпускает лучи из движущегося сенсора, учитывает ближайшие пересечения и окклюзии, формирует облако точек с характерными зонами видимости/тени и сохраняет совместимость с текущими semantic class, RGB, конфигурацией и metadata.
