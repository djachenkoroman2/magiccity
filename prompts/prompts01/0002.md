Да, можно. Принципы генерации Minecraft хорошо подходят для синтетических ландшафтных облаков точек, только вместо блоков ты генерируешь непрерывную поверхность и затем семплируешь ее точками.

Главное отличие:

```text
Minecraft:
noise/density -> блоки stone/air/water/grass

Point cloud:
noise/density -> поверхность -> точки x,y,z + цвет/класс/нормаль
```

**Какой Подход Выбрать**

Для простых ландшафтов достаточно heightmap:

```text
height = f(x, z)
point = (x, height, z)
```

Так удобно делать равнины, холмы, горы, берега, океанское дно.

Для Minecraft-подобного мира с пещерами, арками, нависающими скалами лучше использовать 3D density field:

```text
density = f(x, y, z)

density > 0 => земля/камень
density <= 0 => воздух
```

А потом извлекать поверхность `density = 0` и рассыпать по ней точки.

**Пайплайн**

1. **Seed**
   
   Один seed задает весь мир:

```text
seed + координаты => повторяемый ландшафт
```

2. **Крупные шумы**
   
   Генерируешь материки, горы, долины, океаны:

```text
continentalness = noise(x * 0.001, z * 0.001)
erosion         = noise(x * 0.003, z * 0.003)
weirdness       = noise(x * 0.006, z * 0.006)
```

3. **Высота или плотность**
   
   Heightmap-вариант:

```text
height =
    base_level
  + continentalness * 120
  - erosion * 40
  + mountain_noise * 80
  + detail_noise * 8
```

   Density-вариант:

```text
density(x,y,z) =
    terrain_height(x,z) - y
  + cave_noise(x,y,z)
  + detail_noise(x,y,z)
```

4. **Биомы**
   
   Как в Minecraft, можно считать климатические карты:

```text
temperature = noise(...)
humidity    = noise(...)
```

   И по ним назначать тип поверхности:

```text
холодно + высоко     => снег/камень
жарко + сухо         => песок/пустыня
влажно + тепло       => джунгли
низко + у воды       => берег/песок
```

5. **Surface rules**
   
   Для каждой точки задаешь цвет и класс:

```text
grass  => зеленый
sand   => желтый
rock   => серый
snow   => белый
water  => синий
```

   В point cloud это может стать:

```text
x, y, z, r, g, b, class, normal
```

6. **Семплирование точек**
   
   Если это heightmap:

```text
for x,z over area:
    y = height(x,z)
    emit point(x, y, z)
```

   Если хочешь более натуральное облако, используй jitter/blue noise:

```text
x = grid_x + random_offset
z = grid_z + random_offset
y = height(x,z)
```

   Для 3D density field можно использовать Marching Cubes, получить mesh, а потом равномерно семплировать точки по треугольникам.

7. **Детали**
   
   Отдельно добавляются:

```text
камни
деревья
кусты
трава
реки
эрозионные канавы
обрывы
дороги
пещеры
```

   Например дерево можно сгенерировать процедурно, а затем превратить его поверхность в точки.

8. **LiDAR-стиль**
   
   Если нужна не просто “красивая” точечная поверхность, а реалистичное облако как после сканирования, добавь:

```text
шум дальности
пропуски точек
неравномерную плотность
окклюзии
лучи от позиции сканера
классы ground / vegetation / water / building
```

**Мини-Псевдокод**

```python
for each sample:
    x, z = sample_position()

    climate = get_climate_noise(x, z)
    h = terrain_height(x, z, climate)

    material = surface_material(x, h, z, climate)

    point = {
        "x": x,
        "y": h + small_surface_noise(x, z),
        "z": z,
        "color": material.color,
        "class": material.class_name
    }

    points.append(point)
```

**Вывод**

Да, Minecraft-подход отлично переносится на генерацию ландшафтных облаков точек. Лучше всего использовать его не буквально как “блочный мир”, а как процедурную модель: seed, шумы, климатические карты, density functions, surface rules и features. Потом результат превращается не в воксели, а в облако точек с координатами, цветом, нормалями и классами поверхности.
