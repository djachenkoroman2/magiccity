Ты senior Python-инженер, архитектор процедурной генерации и technical writer для проекта MagicCity.

В репозитории есть рабочий MVP `citygen`: CLI на Python, YAML-конфиги, детерминированный `seed`, тайлы, процедурный рельеф, road primitives, несколько road models, road profiles, worldgen catalogs, urban fields, биомы, parcels, fences, mobile LiDAR, разные building footprints/roofs, surface sampling, ASCII PLY export, metadata JSON и русскоязычная документация в `doc/` с HTML-сборкой через MkDocs. Координатная конвенция проекта: горизонтальная плоскость `x/y`, высота `z`.

Нужно реализовать новый опциональный слой generated objects: деревья. Не ограничивайся описанием подхода: внеси рабочие изменения в код, конфиги, тесты и документацию.

## Главная цель

Добавить в `citygen` независимую подсистему генерации деревьев, которая:

- размещает деревья только на естественном грунте;
- учитывает биомы и плотность посадки;
- поддерживает разные формы кроны;
- параметризуется через YAML;
- участвует в surface sampling и mobile LiDAR;
- экспортирует semantic class/RGB и metadata;
- интегрирована в worldgen pipeline как отдельный понятный этап;
- отражена в Markdown- и HTML-документации.

## Функциональные требования

1. Добавить новую секцию конфигурации `trees`.

   Минимальные ожидаемые параметры:

   - `enabled`: включает или выключает слой деревьев, по умолчанию `false`;
   - `density_per_ha`: базовая плотность посадки деревьев на гектар;
   - `min_spacing_m`: минимальная дистанция между деревьями;
   - `trunk_radius_m`, `trunk_height_ratio`, `height_m`, `height_jitter_m`;
   - параметры кроны: `crown_shape`, `crown_radius_m`, `crown_height_ratio`, `crown_segments`;
   - `weights` для смешанного выбора форм кроны;
   - `biome_density_multipliers` или аналогичный механизм для разных биомов;
   - параметры clearance от дорог, зданий, fences и границ тайла;
   - `sample_spacing_m` для плотности точек дерева.

   Поддержи строгую валидацию секции `trees`: неизвестные поля должны давать понятную ошибку.

2. Поддержать формы крон.

   Минимальный набор:

   - `round` или `sphere`: округлая крона;
   - `ellipsoid`: вытянутая овальная крона;
   - `cone`: хвойная коническая крона;
   - `columnar`: узкая вертикальная крона;
   - `umbrella`: широкая зонтичная крона.

   Для каждой формы должны работать общие параметры высоты, радиуса, высоты кроны и детерминированного jitter. Если вводятся aliases, опиши их в catalogs/docs и покрой тестами.

3. Размещение деревьев должно учитывать поверхность и занятость сцены.

   Деревья можно ставить только на natural ground:

   - не на дорогах;
   - не на тротуарах;
   - не на road medians, если явно не добавлен отдельный параметр для medians;
   - не внутри footprints зданий;
   - не в зоне clearance вокруг зданий;
   - не на fences/fence foundations;
   - не за пределами crop bbox тайла;
   - учитывать рельеф через `terrain_height`.

   Алгоритм размещения должен быть детерминированным от `seed`, координат тайла, биома и локального id дерева.

4. Плотность должна зависеть от биома.

   Сделай реалистичные значения по умолчанию:

   - `suburb`: высокая плотность;
   - `residential`: средняя плотность;
   - `downtown`: низкая плотность;
   - `industrial`: очень низкая плотность или отсутствие деревьев.

   Должна быть возможность явно задать `0` для биома, где деревьев быть не должно.

5. Архитектура должна быть модульной.

   Добавь независимый модуль генерации, например `citygen/trees.py`, не раздувая `generator.py` и `sampling.py` сверх необходимости.

   Ожидаемые сущности:

   - dataclass для экземпляра дерева;
   - dataclass/объект для спецификации кроны;
   - функция генерации деревьев по scene context;
   - функция семплирования trunk/crown surfaces;
   - helpers для пересечений mobile LiDAR, если нужен отдельный код.

   Новая подсистема должна быть встроена в существующий pipeline, но не должна ломать существующие сцены при `trees.enabled: false`.

6. Интегрировать деревья в пайплайн.

   Нужно добавить отдельный stage, например `trees`, между размещением buildings/parcels/fences и sampling, если это соответствует текущей архитектуре.

   Обнови:

   - worldgen catalogs/stages;
   - CLI/progress diagnostics, если они перечисляют стадии;
   - metadata pipeline summary;
   - документацию с визуализацией или перечнем стадий всего pipeline.

   Цель: пользователь должен видеть, что генерация деревьев является отдельным этапом, а не скрытой частью sampling.

7. Семантические классы и RGB.

   Добавь классы для дерева, не меняя существующие class ids:

   - `tree_trunk`;
   - `tree_crown`.

   Назначь стабильные новые ids и RGB-цвета. Обнови class mapping, metadata, docs и тесты. Существующие semantic class ids менять нельзя.

8. Sampling.

   Surface sampling должен генерировать точки:

   - на стволе;
   - на поверхности кроны;
   - с корректным `z`, основанным на рельефе в точке посадки;
   - с semantic class и RGB;
   - с контролируемой плотностью через `trees.sample_spacing_m`.

   Не допускай чрезмерного роста количества точек на больших тайлах: используй разумные defaults и ограничения.

9. Mobile LiDAR.

   Если mobile LiDAR включен, лучи должны уметь попадать в деревья:

   - стволы должны работать как цилиндрические препятствия;
   - кроны должны давать попадания по форме кроны;
   - ближайшее попадание по лучу должно учитывать деревья вместе с terrain, roads, buildings и fences;
   - при включенных occlusions дерево может закрывать объекты позади.

   Если полная физика крон слишком велика для одного изменения, реализуй минимальный корректный вариант и явно зафиксируй ограничение в документации.

10. Metadata.

    Добавь диагностические поля:

    - общее число деревьев;
    - counts по форме кроны;
    - counts по биому;
    - средняя/минимальная/максимальная высота;
    - количество trunk/crown points;
    - параметры `trees` после применения defaults;
    - участие деревьев в `object_feature_counts`.

11. Demo-конфиги.

    Добавь или обнови demo-конфиг:

    - либо отдельный `configs/demo_trees.yaml`;
    - либо аккуратное включение деревьев в `configs/demo_universal_showcase.yaml`, если это не делает showcase слишком тяжелым.

    В демо должны быть видны разные формы крон и разные плотности по биомам.

12. Документация.

    Обнови Markdown-документацию на русском языке:

    - `doc/configuration_reference.md`;
    - отдельный тематический документ, например `doc/trees.md`;
    - `doc/generated_objects.md`;
    - `doc/sampling.md`;
    - `doc/worldgen_catalogs.md`;
    - `doc/universal_showcase.md`, если showcase изменен;
    - `README.md` и `configs/README.md`, если добавлен новый demo.

    Обнови `mkdocs.yml` и пересобери HTML:

    ```bash
    uv run mkdocs build --clean
    ```

    В результате `doc/html/` должен содержать актуальную HTML-документацию, включая страницу про деревья.

## Тесты

Добавь тесты, которые проверяют:

- загрузку и defaults секции `trees`;
- валидацию отрицательных плотностей, размеров, spacing и неизвестных форм крон;
- `trees.enabled: false` не меняет старое поведение;
- при включенных деревьях появляются экземпляры деревьев и точки `tree_trunk`/`tree_crown`;
- деревья не размещаются на дорогах, тротуарах и зданиях;
- biome density multipliers реально влияют на количество деревьев;
- одинаковый `seed` дает одинаковые деревья и точки;
- metadata содержит tree counts и class mapping;
- mobile LiDAR, если включен, может попадать в деревья или корректно документирует временное ограничение.

Минимально запусти:

```bash
uv run python -m unittest discover -s tests
uv run mkdocs build --clean
git diff --check
```

Если генерация полного showcase тяжелая, используй маленький тестовый конфиг или временный output в `/tmp`, но проверь, что новый layer действительно работает.

## Ограничения и совместимость

- Не меняй существующие semantic class ids.
- Не ломай существующие YAML-конфиги.
- Не делай деревья включенными по умолчанию, если это меняет существующие outputs.
- Не размещай деревья на искусственных surfaces без явного параметра.
- Не добавляй внешние тяжелые зависимости без необходимости.
- Не подменяй реализацию только документацией: новый слой должен реально генерировать объекты и точки.

## Ожидаемый результат

Итоговая реализация должна добавить реалистичный, конфигурируемый и детерминированный слой деревьев: разные формы крон, biome-aware density, размещение только на естественном грунте, корректные semantic classes, sampling, metadata, pipeline stage, demo-конфиг, тесты и актуальную Markdown/HTML-документацию.
