# Рельеф

`terrain` задает процедурную функцию высоты поверхности. Все расчеты идут в координатах проекта: горизонтальная плоскость `x/y`, высота `z`.

Высота в точке складывается из базовой высоты, детерминированного шума, поднятий гор/холмов и вычитания оврагов:

```text
z = base_height_m + noise + mountains + hills - ravines
```

Эта функция используется для точек земли, дорог, тротуаров, road medians, базовой высоты зданий, оснований fences и пересечений лучей mobile LiDAR с землей.

## YAML

```yaml
terrain:
  base_height_m: 0
  height_noise_m: 6
  mountains:
    - center_x: 210
      center_y: 230
      height_m: 180
      radius_m: 260
  hills:
    - center_x: 930
      center_y: 260
      height_m: 34
      radius_m: 210
  ravines:
    - center_x: 690
      center_y: 450
      length_m: 760
      width_m: 110
      depth_m: 42
      angle_degrees: 34
```

## Поля

| Параметр | Тип | По умолчанию | Валидация | Действие |
| --- | --- | --- | --- | --- |
| `base_height_m` | number | `0.0` | любое число | Смещает весь рельеф по оси `z`. |
| `height_noise_m` | number | `1.5` | `>= 0` | Амплитуда базового процедурного шума. `0` отключает только шум. |
| `mountains` | list | `[]` | список объектов поднятия | Высокие радиальные поднятия с более резким профилем. |
| `hills` | list | `[]` | список объектов поднятия | Мягкие радиальные поднятия. |
| `ravines` | list | `[]` | список объектов оврага | Линейные понижения с гладким поперечным профилем и затуханием на концах. |

Секция `terrain` валидирует имена параметров строго: неизвестное поле считается ошибкой конфига.

## Горы и холмы

Элементы `mountains` и `hills` имеют одинаковую схему:

| Поле | Тип | Обязательное | Валидация | Действие |
| --- | --- | --- | --- | --- |
| `center_x` | number | да | любое число | X-координата центра поднятия. |
| `center_y` | number | да | любое число | Y-координата центра поднятия. |
| `height_m` | number | да | `> 0` | Максимальная добавочная высота в центре. |
| `radius_m` | number | да | `> 0` | Радиус влияния поднятия. За радиусом вклад равен нулю. |

`mountains` используют более резкий falloff, поэтому подходят для высоких горных массивов и пиков. `hills` используют более мягкий falloff и лучше подходят для плавных холмов.

## Овраги

Элементы `ravines` описывают повернутую линейную впадину:

| Поле | Тип | Обязательное | По умолчанию | Валидация | Действие |
| --- | --- | --- | --- | --- | --- |
| `center_x` | number | да | нет | любое число | X-координата центра оврага. |
| `center_y` | number | да | нет | любое число | Y-координата центра оврага. |
| `length_m` | number | да | нет | `> 0` | Длина центральной части оврага вдоль оси. |
| `width_m` | number | да | нет | `> 0` | Полная ширина оврага поперек оси. |
| `depth_m` | number | да | нет | `> 0` | Максимальная глубина в центральной линии. |
| `angle_degrees` | number | нет | `0.0` | любое число | Поворот оси оврага в градусах. |

При `angle_degrees: 0` овраг вытянут вдоль оси `x`. Положительные значения поворачивают его против часовой стрелки в плоскости `x/y`.

## Showcase-конфиг

`configs/demo_universal_showcase.yaml` содержит пример с двумя высокими горами, тремя холмами и двумя диагональными оврагами. Это удобно для проверки того, что рельеф виден в земле, дорожных поверхностях, основаниях зданий и LiDAR-пересечениях.
